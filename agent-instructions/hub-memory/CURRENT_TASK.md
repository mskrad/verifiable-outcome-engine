- `HACKATHON-FORMULA-001` architect pass is complete and ready for Hub decision on 2026-04-30. The hard compatibility fact is that current W3O1 cannot carry `formula`, `score`, or `target` directly without Rust/Anchor changes: the TS serializer and Rust parser only accept format versions `1` and `2`, require reserved bytes to remain zero, and reject trailing bytes. The selected design is additive: keep W3O1 v1/v2 unchanged, bind a canonical formula-manifest sidecar through `audit_hash` + `artifact_uri`, and compile deterministic formulas (`rank_desc`, `rank_asc`, `first_n`, `closest_to`) into ordinary W3O1 weights before submit so the unchanged weighted-random on-chain resolver lands on the intended formula winner set. Replay then validates the sidecar hash, recomputes the formula winner order, recompiles the expected W3O1 blob from replay randomness, and compares that compiled hash plus winner set against the on-chain result. `weighted_random` stays on the current path; `closest_to` tie-break is stable input order after absolute-distance sort.
