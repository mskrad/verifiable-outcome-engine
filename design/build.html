How it works

Three steps. Three commits. Zero hidden state.

Each step produces a public artifact. Each artifact references the previous one. You never need to trust the engine — the chain of hashes is the proof.

Steps 1–2 are for operators. Step 3 is for everyone.

1

Define rules

Write a JSON config: participants, weights, outcome ids, selection mode. The SDK hashes it and gives you an artifact_hash.

$ vre init raffle.json
2

Commit on-chain

Push the hash to Solana. The VRE program stores it against a runtime_id. Once committed, the ruleset cannot be edited.

$ vre resolve --config raffle.json --program-id <ID>
3

Share and verify

You get a signature. Paste it at verifiableoutcome.online/verify or run vre verify. Anyone does this — not just you.

$ vre verify --sig <SIG> → MATCH / OK
Install

One npm package. Three commands.

~/projects/my-dapp
# Global CLI — for scripting, CI, and manual resolves $ npm install -g verifiable-outcome-sdk # Or embed the programmatic API in your dApp $ npm install verifiable-outcome-sdk # The same deterministic kernel runs in Node, browser, and Deno $ vre --version verifiable-outcome-sdk v1.0.0 kernel: vre-core@1.0.0 (sha256 committed)
raffle.json
// The shape VRE commits on-chain { "version": "1.0", "runtime_id": "rt_8a1f…c4d2", "ruleset_id": "rs_2e7b…9f01", "mode": "raffle", "participants": [ "7FHkp…3GnV", "9LxTm…bV2a", "4NpQr…X9eY" ], "weights": [5, 3, 2], "seed_fn": "slot_nonce" }
Programmatic API
// The same engine, from JS import { VRE } from "verifiable-outcome-sdk"; const vre = new VRE({ cluster: "devnet", programId: "3b7T…XkBq", }); const runtime = await vre.commit(config); const outcome = await vre.resolve(runtime.id); // Independent check — anyone can run this const result = await vre.verify(outcome.sig); // { status: "MATCH", code: "OK", … }

Use cases

Committed rules, single or multi-winner outcomes.

The SDK shape stays the same: build the artifact, resolve on-chain, then replay the transaction signature.

🎟

Raffle / Airdrop

Pick one recipient from a committed weighted address list.

🏆

Multi-Winner

Select N distinct winners in one resolve. Replay returns the primary outcome_id plus the full outcome_ids list.

🗳️

DAO Proposal

Choose from approved proposals after weights are committed.

🎁

Loot

Resolve weighted outcome tables with the same deterministic replay path.
