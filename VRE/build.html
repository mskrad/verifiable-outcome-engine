Build with VRE

On-chain proof of who won. Build committed rules, resolve once, let anyone verify from a transaction signature.

How it works

Three steps. Three commits. Zero hidden state.

Each step produces a public artifact referencing the previous one. The chain of hashes is the proof — no trust in the engine required. Steps 1–2 are for operators. Step 3 is for everyone.

1
commit

Define rules

Write a JSON config: participants, weights, outcome ids, selection mode. The SDK hashes it and gives you an artifact_hash.

$ vre init raffle.json
2
resolve

Commit on-chain

Push the hash to Solana. The VRE program stores it against a runtime_id. Once committed, the ruleset cannot be edited.

$ vre resolve --config raffle.json
3
verify

Share & verify

You get a signature. Paste it at /verify or run vre verify. Anyone does this — not just you.

$ vre verify --sig <SIG> → MATCH / OK

The shape

Config in. Signature out. Replay anywhere.

raffle.jsonconfig
// The shape VRE commits on-chain { "version": "1.0", "runtime_id": "rt_8a1f…c4d2", "ruleset_id": "rs_2e7b…9f01", "mode": "raffle", "participants": [ "7FHkp…3GnV", "9LxTm…bV2a", "4NpQr…X9eY" ], "weights": [5, 3, 2], "seed_fn": "slot_nonce" }
app.tsprogrammatic
// Same engine, from JS import { VRE } from "verifiable-outcome-sdk"; const vre = new VRE({ cluster: "devnet", programId: "9tEram…FE1F", }); const runtime = await vre.commit(config); const outcome = await vre.resolve(runtime.id); // Independent check — anyone can run this const result = await vre.verify(outcome.sig); // { status: "MATCH", code: "OK", … }

Use cases

Committed rules. Single or multi-winner outcomes.

Same SDK shape: build artifact, resolve on-chain, replay the signature.

Raffle

W3O1 v1 weighted random. One winner from a committed weighted participant list.

Rewards Selection

W3O1 v2 multi-winner weighted random. Replay returns full ordered outcome_ids.

Trading Competition

W3O1 v3 rank_desc. Signed scores and stable input order produce deterministic top-N winners.

Prediction Market

W3O1 v3 closest_to. Replay includes committed target, signed scores, and stable order.
