Config Engine + SDK

Build with VRE

Configure verifiable raffles, loot tables, and airdrops. Commit the rules on-chain before the outcome is drawn, then let anyone replay the result from public RPC data.

The npm SDK builds artifacts, resolves against an existing deployed program, and verifies outcomes. The SDK does not deploy the Solana program. To operate your own program instance, clone the repo, deploy programs/outcome with Anchor, then use the SDK/CLI with --program-id.

Use Cases

🎟

Raffle

Select a winner from a weighted participant list with Solana addresses as outcome ids.

🎁

Loot

Roll a weighted outcome table for legendary, rare, and common reward paths.

🪂

Airdrop

Weighted eligible-list selection from an address set. Multi-winner support is experimental.

🔮

Prediction Market

Pre-commit possible outcomes and their weights before the event resolves. Anyone can verify the declared result matches the pre-committed spec.

How It Works

1
Define rules

Create a JSON config with participants, weights, payout bounds, or outcome ids.

2
Commit on-chain

Run vre resolve --config raffle.json --program-id <PROGRAM_ID> to publish the artifact to an existing program and resolve a transaction.

3
Share and verify

Anyone pastes the transaction at verifiableoutcome.online and gets MATCH / OK plus pre-commitment proof.

SDK

npm install -g verifiable-outcome-sdk 
vre verify --sig <SIGNATURE>

# Or without a global install:
npx -p verifiable-outcome-sdk vre verify --sig <SIGNATURE>
import { buildArtifact, verifyOutcome } from 'verifiable-outcome-sdk';

// Define your raffle
const blob = buildArtifact({
  type: 'raffle',
  input_lamports: 10,
  participants: [
    { address: '5RbvSHb...', weight: 1000 },
    { address: 'Aip3wC6...', weight: 1000 },
  ],
});

// Verify any resolved TX
const result = await verifyOutcome({
  signature: 'mUXwae...',
  rpcUrl: 'https://api.devnet.solana.com',
});
// -> { status: 'MATCH', outcome_id: '5RbvSHb...', ... }

The executable is vre, not verifiable-outcome-sdk. Program owners clone the repo, deploy programs/outcome with Anchor, then use this SDK against their program id.
