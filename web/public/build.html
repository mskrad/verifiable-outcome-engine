Config Engine + SDK

Build with VRE

Configure verifiable raffles, loot tables, and airdrops. Commit the rules on-chain before the outcome is drawn, then let anyone replay the result from public RPC data.

Use Cases

🎟

Raffle

Select a winner from a weighted participant list with Solana addresses as outcome ids.

🎁

Loot

Roll a weighted outcome table for legendary, rare, and common reward paths.

🪂

Airdrop

Select N winners from an eligible address list; duplicate winners are allowed for now.

How It Works

1
Define rules

Create a JSON config with participants, weights, payout bounds, or outcome ids.

2
Commit on-chain

Run yarn resolve:operator --config raffle.json to publish the artifact and resolve a transaction.

3
Share and verify

Anyone pastes the transaction at verifiableoutcome.online and gets MATCH / OK plus pre-commitment proof.

SDK

npm install verifiable-outcome-sdk 
import { buildArtifact, verifyOutcome } from 'verifiable-outcome-sdk';

// Define your raffle
const blob = buildArtifact({
  type: 'raffle',
  input_lamports: 10,
  participants: [
    { address: '5RbvSHb...', weight: 1000 },
    { address: 'Aip3wC6...', weight: 1000 },
  ],
});

// Verify any resolved TX
const result = await verifyOutcome({
  signature: 'mUXwae...',
  rpcUrl: 'https://api.devnet.solana.com',
});
// -> { status: 'MATCH', outcome_id: '5RbvSHb...', ... }
