1. Rules

A ruleset is a JSON object hashed into a 32-byte artifact. The hash is committed on-chain; the full object lives anywhere public (IPFS, R2, GitHub). Fields:

FieldTypeMeaning
versionstringSemver. Currently "1.0".
runtime_idstringCaller-chosen identifier. Must be unique per program.
ruleset_idstringCanonical hash-derived ID: rs_<first 8 of sha256>….
modeenumraffle | airdrop | prediction | loot
participantsstring[]Ordered list of addresses or outcome IDs. Case-sensitive.
weightsnumber[]Same length as participants. Must be ≥ 0.
seed_fnenumslot_nonce | block_hash | vrf
outputsnumberNumber of winners to select. Default 1.

2. Input Schema

JSON-schema validation applied at vre commit. Extra fields are preserved in the artifact but ignored by the kernel.

schema.json
{ "$schema": "https://json-schema.org/draft/2020-12/schema", "type": "object", "required": ["version", "mode", "participants", "weights"], "properties": { "version": { "const": "1.0" }, "mode": { "enum": ["raffle","airdrop","prediction","loot"] }, "participants": { "type": "array", "items": { "type": "string" } }, "weights": { "type": "array", "items": { "minimum": 0 } }, "seed_fn": { "enum": ["slot_nonce","block_hash","vrf"] }, "outputs": { "type": "integer", "minimum": 1 } } }

3. Determinism

The selection function is a pure mapping f(seed, rules) → outcome. Three guarantees:

  • Referential transparency: same inputs always yield the same output, forever.
  • No hidden state: the kernel reads nothing except what's in the ruleset and the seed.
  • Platform-neutral: Node, browser, Deno, Rust, Python all produce bit-identical results.
The kernel source is vre-core. Its sha256 is committed on-chain alongside every resolve. A kernel upgrade is a new hash — old outcomes never change.

4. Replay Process

Given a transaction signature, the verifier performs:

  1. Fetch the transaction from any Solana RPC endpoint.
  2. Confirm it invokes the VRE program and is a resolve instruction.
  3. Extract runtime_id, artifact_hash, seed, claimed winner.
  4. Fetch the referenced commit transaction; confirm its slot < resolve slot.
  5. Re-hash the ruleset and confirm it matches the committed artifact_hash.
  6. Run f(seed, rules) locally; compare to the claimed winner.

If all six steps pass: MATCH / OK. Otherwise, a specific error code — never a silent pass.

Protocol Fee

Each resolveOutcome call may collect a protocol fee from the calling operator. The transfer happens before resolution; if the operator wallet cannot pay, the transaction fails and no outcome is recorded.

Current devnet fee: 0 lamports. The value is configurable through ProgramConfig.fee_lamports by the program admin without a program upgrade.

Agent-to-agent use

Any agent that selects a winner, assigns a task, or picks a DAO proposal can call POST /api/replay with the resolve transaction signature. VRE replays the outcome from public RPC and returns MATCH / OK — or an error code if the result was tampered with. No oracle, no operator trust required.

# Any agent verifies a VRE outcome $ curl -s https://verifiableoutcome.online/api/replay \ -H "content-type: application/json" \ -d '{"signature":"<resolve_tx_sig>","programId":"3b7TFKQWUhPqWBieLHop4Mj2e41vwvnvjEosbsdmXkBq"}' → { "ok": true, "replay": { "verification_result": "MATCH", "verification_reason": "OK", … } }

9. Private Settlement via Vanish

VRE proves the winner was selected without manipulation — on-chain and replayable. Vanish routes the payout so the operator's treasury strategy stays private. Verifiable selection and private settlement — neither is possible alone.

How it works
  1. VRE selects the winner publicly — outcome committed on-chain, replayable by anyone.
  2. Operator calls Vanish /deposit_address and transfers SOL to the Vanish pool.
  3. Vanish routes funds to winner from an anonymous one-time address — on-chain link between operator treasury and winner wallet is broken.
  4. Both the VRE resolve sig (public) and Vanish withdraw tx (private routing) are written to artifacts/vanish_integration_evidence.json.
LayerWhat it provesPublic?
VRE resolve tx Winner was selected by a deterministic, tamper-evident algorithm Yes — verifiable by anyone
Vanish payout tx Winner received SOL from a Vanish one-time address — no direct link to operator treasury On-chain but unlinked
Optional env vars
VANISH_API_KEY Vanish Core API key. When set, live-raffle automatically routes payout via Vanish.
VANISH_PAYOUT_LAMPORTS Amount to route per payout. Default: 100000 (0.0001 SOL).
Standalone CLI
# Route a payout after a known VRE resolve tx $ VANISH_MOCK=true ts-node scripts/vanish_payout.ts \ --winner <winner_address> \ --amount 100000 \ --sig <vre_resolve_sig> → artifacts/vanish_integration_evidence.json
